LibreOffice 5.4 ReadMe
======================================================================


Актуальная версия файла readme: http://www.libreoffice.org/welcome/readme.html

Этот файл содержит важную информацию о LibreOffice. Перед началом установки рекомендуется внимательно прочесть эту информацию.

Сообщество LibreOffice отвечает за разработку этого продукта и предлагает вам рассмотреть возможность вашего участия в качестве члена сообщества. Если вы новичок, вы можете посетить сайт LibreOffice, где вы найдёте множество информации о проекте LibreOffice и соседних сообществах. Посетите http://www.libreoffice.org/.

Действительно ли LibreOffice свободен для всех пользователей?
----------------------------------------------------------------------

LibreOffice бесплатен для всех. Вы можете взять эту копию LibreOffice и установить её на любое, нужное вам, количество компьютеров и использовать его в любых целях (включая коммерческие, государственные, административные и образовательные). Более подробную информацию см. в тексте лицензионного соглашения идущего в комплекте с LibreOffice.

Почему LibreOffice бесплатен для всех пользователей?
----------------------------------------------------------------------

Вы можете бесплатно использовать LibreOffice, потому что индивидуальные разработчики и корпоративные спонсоры спроектировали, разработали, протестировали, перевели, задокументировали, поддерживали, распространяли и помогали многими другими способами, чтобы сделать LibreOffice тем, чем он является сегодня - мировым лидером, эффективным Open Source программным обеспечением для дома и офиса.

Если вы цените их усилия, и хотите быть уверенными в том, что LibreOffice будет доступен в далёком будущем, пожалуйста, задумайтесь о поддержке проекта - подробнее см. http://www.documentfoundation.org/contribution/. Каждый может внести свой вклад.

----------------------------------------------------------------------
Замечания по установке
----------------------------------------------------------------------

LibreOffice требуется последняя версия Java Runtime Environment (JRE) для полной функциональности. JRE не является частью установочного пакета LibreOffice и требует отдельной установки.

Системные требования
----------------------------------------------------------------------

* Microsoft Windows XP SP3, Vista, 7, 8 или 10

Примечание: для выполнения установки необходимы права администратора.

Разрешить или запретить регистрацию LibreOffice как основного приложения для форматов Microsoft Office можно при помощи следующих параметров командной строки программы установки:

* REGISTER_ALL_MSO_TYPES=1 зарегистрировать LibreOffice как основное приложения для форматов Microsoft Office.
* REGISTER_NO_MSO_TYPES=1 не регистрировать LibreOffice как основное приложения для форматов Microsoft Office.

Убедитесь, пожалуйста, что в вашей системе имеется достаточное количество свободной памяти во временной директории и, пожалуйста, убедитесь в том, что установлены права на чтение, запись и выполнение. Закройте все другие программы перед началом установки.

Установка LibreOffice в Debian/Ubuntu-подобных Linux системах
----------------------------------------------------------------------

Инструкция по установке языкового пакета (поверх установленной английской (US English) версии LibreOffice) находится в разделе Установка языкового пакета.

При распаковке загруженного архива его содержимое распакуется в некоторый подкаталог. Откройте окно файлового менеджера и измените каталог, начинающийся с "LibreOffice_", за которым следует номер версии и информация о платформе.

Директория содержит поддиректорию с именем "DEBS". Перейдите в директорию "DEBS".

Кликните правой кнопкой мыши в директории и выберите "Открыть в терминале". Откроется окно терминала. В командной строке терминала введите следующую команду (прежде чем команда будет выполнена, вам будет предложено ввести пароль суперпользователя):

Следующие команды установят пакеты LibreOffice и интеграции в рабочее окружение (можно просто скопировать и вставить их в окно терминала):

sudo dpkg -i *.deb

Теперь процесс установки завершён и в системном меню Приложения/Офис должны появиться иконки для запуска приложений LibreOffice.

Установка LibreOffice в Fedora, openSUSE, Mandriva и другие системы Linux с использованием RPM-пакетов
----------------------------------------------------------------------

Инструкция по установке языкового пакета (поверх установленной английской (US English) версии LibreOffice) находится в разделе Установка языкового пакета.

При распаковке загруженного архива его содержимое распакуется в некоторый подкаталог. Откройте окно файлового менеджера и измените каталог, начинающийся с "LibreOffice_", за которым следует номер версии и информация о платформе.

Директория содержит поддиректорию с именем "RPMS". Перейдите в директорию "RPMS".

Кликните правой кнопкой мыши в директории и выберите "Открыть в терминале". Откроется окно терминала. В командной строке терминала введите следующую команду (прежде чем команда будет выполнена, вам будет предложено ввести пароль суперпользователя):

Для Fedora-подобных систем: su -c 'yum install *.rpm'

Для Mandriva-подобных систем: sudo urpmi *.rpm

Для других систем, основанных на RPM (openSUSE и др.): rpm -Uvh *.rpm

Теперь процесс установки завершён и в системном меню Приложения/Офис должны появиться иконки для запуска приложений LibreOffice.

В качестве альтернативы можно использовать 'install' скрипт, размещённый в каталоге верхего уровня архива для установки от имени пользователя. Скрипт установит LibreOffice со своим собственным профилем, отдельно от обычного LibreOffice профиля. При этом не будет выполнена интеграция в систему: элементы меню и регистрация типов MIME.

Примечания, касающиеся интеграции с рабочим столом тех дистрибутивов Linux, для которых не описан процесс установки в инструкциях выше
----------------------------------------------------------------------

Можно легко установить LibreOffice на других Linux системах, не указанных в этой инструкции по установке. Главное различие заключается в интеграции с рабочим столом.

Каталог RPMS (или DEBS) также содержит пакет 5.4-freedesktop-menus-5.4.0.1-1.noarch.rpm (libreoffice5.4-debian-menus_5.4.0.1-1_all.deb или подобный). Это пакет для всех систем Linux, поддерживающих спецификации и рекомендации Freedesktop.org (http://ru.wikipedia.org/wiki/Freedesktop.org), и предназначен для установки на других Linux системах, не охваченных в вышеупомянутых инструкциях.

Установка языкового пакета
----------------------------------------------------------------------

Загрузите языковой пакет с нужным языком и для необходимой платформы. Языковые пакеты находятся в том же месте, что и главный установочный архив. С помощью файлового менеджера Nautilus распакуйте архив в директорию (например, на рабочий стол). Убедитесь, что вы вышли из всех приложений LibreOffice (включая QuickStarter).

Перейдите в каталог, в который распаковали загруженный языковой пакет.

Теперь перейдите в каталог, который был создан в процессе распаковки. Например, для французского языкового пакета для 32-bit Debian/Ubuntu-based system, каталог будет называться LibreOffice_, плюс информация о версии, плюс Linux_x86_langpack-deb_fr.

Теперь перейдите в каталог, который был создан в процессе извлечения. Для систем, основанных на Debian/Ubuntu-based, каталог будет называться DEBS. Для Fedora, openSUSE или Mandriva каталог будет называться RPMS.

В файловом менеджере Nautilus нажмите правой кнопкой мыши в директории и выберите команду "Открыть в терминале". В открывшемся окне терминала выполните команду для установки языкового пакета (для всех нижеперечисленных команд может потребоваться ввод пароля суперпользователя):

Для Debian/Ubuntu-подобных систем: sudo dpkg -i *.deb

Для Fedora-подобных систем: su -c 'yum install *.rpm'

Для Mandriva-подобных систем: sudo urpmi *.rpm

Для других RPM-систем (openSUSE и др.): rpm -Uvh *.rpm

Теперь запустите одно из LibreOffice приложений, например, Writer. Перейдите в меню Сервис - Параметры. В диалоговом окне Параметры, кликните на "Настройки языка", а затем на "Языки". В выпадающем списке "Пользовательский интерфейс" выберите язык, который только что установили. Если нужно, сделайте тоже самое для "Локальные настройки", "Валюта по умолчанию" и для "Языки документов по умолчанию".

После установки этих параметров нажмите ОК. Диалоговое окно закроется и вы увидите сообщение о том, что ваши изменения будут применены после выхода из LibreOffice и следующего запуска (не забудьте также выйти из QuickStarter если он запущен).

При следующем запуске LibreOffice запустится с только что установленным языком.

Проблемы при запуске программы
----------------------------------------------------------------------

Трудности при запуске LibreOffice (например, зависание приложений), а также проблемы с экранным отображением часто вызваны драйвером видеокарты. При возникновении этих проблем обновите драйвер или используйте драйвер, включённый в состав используемой операционной системы. Для решения проблем при отображении трёхмерных объектов следует отключить параметр «Использовать OpenGL» в меню «Сервис - Параметры - LibreOffice - Вид - Графический вывод».

Сенсорные панели ноутбуков ALPS/Synaptics в Windows
----------------------------------------------------------------------

В связи с проблемой драйвера Windows вы не сможете прокручивать документы LibreOffice, водя пальцем по сенсорной панели ALPS/Synaptics.

Чтобы включить прокрутку с помощью сенсорных панелей, добавьте следующие строки в файл конфигурации "C:\Program Files\Synaptics\SynTP\SynTPEnh.ini" и перезагрузите компьютер:

[LibreOffice]

FC = "SALFRAME"

SF = 0x10000000

SF |= 0x00004000

Замечание: расположение конфигурационного файла может отличаться в разных версиях Windows.

----------------------------------------------------------------------
Комбинации клавиш
----------------------------------------------------------------------

Только комбинации клавиш, не используемые операционной системой, могут быть задействованы в LibreOffice. Если комбинации клавиш в LibreOffice не действуют, как это описано в справке, проверьте не используются ли они уже операционной системой. Для разрешения подобных конфликтов Вы можете изменить комбинации, используемые операционной системой. Вы можете также изменить почти все комбинации для LibreOffice. За дополнительной информацией по этому вопросу обращайтесь к справочному руководству LibreOffice или документации операционной системы.

----------------------------------------------------------------------
Проблемы при отправке документов как электронных писем из LibreOffice
----------------------------------------------------------------------

При отправке документа посредством 'Файл - Отправить - Документ электронной почтой' или 'Электронной почтой как PDF' возможно аварийное завершение или «зависание» программы. Это происходит из-за входящего в Windows системного файла «Mapi» (Messaging Application Programming Interface), некоторые версии которого вызывают проблемы. К сожалению, решение не может быть сведено к указанию правильного номера версии этого файла. Для дополнительной информации посетите http://www.microsoft.com и выполните поиск в Базе знаний Microsoft по «mapi dll».
